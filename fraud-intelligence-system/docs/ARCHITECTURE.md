+﻿# Architecture
+
+## Overview
+The Fraud Intelligence System is a synthetic fraud scoring pipeline composed of:
+- A FastAPI service for transaction scoring and sample generation.
+- A synthetic data generator used for training and demo traffic.
+- A lightweight PyTorch model with a simple feature engineering pipeline.
+
+## Components
+
+### API Layer (`api/fraud_api.py`)
+- Exposes `GET /health`, `GET /transactions/generate`, and `POST /transactions/score`.
+- Uses synthetic transactions as a base payload to ensure all expected fields exist.
+- Returns a `risk_score`, a binary `is_fraud` flag, and basic reasons.
+- Raises a 503 if model artifacts are missing.
+
+### Data Generation (`data/transaction_generation.py`)
+- Generates realistic-looking synthetic transaction records.
+- Produces a synthetic label with heuristic rules for training.
+- Includes helper functions for “investigation” summaries and suggested actions.
+
+### Model Training (`model/train.py`)
+- Builds numeric + one-hot categorical features.
+- Trains a simple feed-forward network (`FraudNet`).
+- Saves model weights and the training feature columns.
+
+### Inference (`model/infer.py`)
+- Loads the saved model weights and feature columns.
+- Aligns incoming payloads to the training feature space.
+- Returns a probability score from the model.
+
+## Data Flow
+1. Client submits a transaction to `/transactions/score`.
+2. API merges it with a synthetic base payload (ensures all fields exist).
+3. Model inference converts the payload into features and scores it.
+4. API returns score + a simple rules-based explanation.
+
+## Artifacts
+- `model/fraud_net.pt`: trained PyTorch weights
+- `model/feature_columns.json`: feature ordering from training
+
+These are generated by `python model/train.py` and required for inference.
